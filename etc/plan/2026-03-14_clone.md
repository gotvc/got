# got clone


## Goal
```
got clone --mkvol <new-vol-name> \
  --url <url> \
  --secret <secret>
```

- got clone takes the same arguments as got init 
- got clone does the same thing as got init, except it additionally configures the repository and downloads data.
- Cloning should additinally:
  - Create a new space in the repo called `origin`, the spec for this is taken from --url and --secret.
  - Setup a fetch task which will sync all of the origin's branches into "remote/origin/"
  - Begin a fetch task.
- It should not perform a checkout.


## Changes
- gotrepo should have a `const OriginSpaceName = "origin"`
- gotrepo should change to have a function `Clone(ctx context.Context, bc blobcache.Env, root *os.Root, spec gotrepo.SpaceSpec)` error
- gotcmd should have a new command `cloneCmd`, which is a subcommand of the root.
- gotcmd: the --url and --secret parameters should be reused.  They handle parsing.
- e2etest: A test TestClone which sites up a site, loads in some data, then creates another site by cloning.
